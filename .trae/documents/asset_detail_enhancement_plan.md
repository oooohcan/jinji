# 资产明细页优化实现计划

## 一、项目研究结论

### 当前状态分析
1. **笔记预测板块**：位于资产明细弹框内，`.prediction-item` 和 `.note-summary-item` 仅使用 `var(--card)` 背景色，无边框无阴影，与面板背景融合度高，视觉区分度不足
2. **资产明细字段**：当前仅展示 6 个字段（成本价、持仓数量、市值、持仓时间、盈亏金额、盈亏比例），缺少持仓列表中已有的关键字段
3. **持仓列表新字段**：三行卡片布局已包含资产类别标签、持仓天数、最新价、今日盈亏、持仓占比等字段，但未同步到明细页
4. **编辑机制**：现有 `enterAssetEditMode()`、`saveAssetInlineEdit()`、`cancelAssetInlineEdit()` 已实现编辑模式切换，支持成本价、数量、持仓天数三个字段编辑

### 涉及文件
- 仅需修改 `/workspace/index.html`（单文件 PWA 应用）

## 二、修改内容

#### CSS 修改

- `.prediction-empty`：保持虚线边框，添加轻微阴影

### 任务 2：扩展资产明细字段
在现有 6 个字段基础上补充持仓列表中的字段，字段布局保持 2 列网格，固收类字段根据资产类型动态显示

#### 新增字段清单
| 字段 | 类型 | 是否可编辑 | 说明 |
|------|------|-----------|------|
| 资产类别 | 展示 | 否 | 显示彩色类别标签（与持仓列表一致） |
| 最新价 | 展示 | 否 | 显示当前最新价格（带货币符号） |
| 今日盈亏 | 展示 | 否 | 金额+收益率，红绿色区分 |

| 到期日 | 固收类专属 | 是 | 存款/理财显示，支持编辑（date 类型输入） |
| 年化收益率 | 固收类专属 | 是 | 存款/理财显示，支持编辑（数字输入） |
| 距到期天数 | 固收类专属 | 否 | 自动计算，只读 |

#### HTML 结构修改
- 在 `.detail-grid` 中添加字段卡片，按逻辑分组排列：
  - 第一行：持仓成本价（已有）、持仓数量（已有）
  - 第二行：最新价（新增）、持仓时间（已有，优化为持仓天数）
  - 第三行：当前总市值（已有）、持仓占比（新增，带进度条）
  - 第四行：今日盈亏（新增）、盈亏金额（已有）
  - 第五行：盈亏比例（已有，占满两列宽）
  - 固收类区域：到期日（可编辑）、年化收益率（可编辑）、距到期天数（只读），通过类名控制显示/隐藏
- 资产类别标签显示在弹框标题区域，位于代码旁边
- 添加对应的编辑输入框：到期日 `<input type="date">`、年化收益率 `<input type="number" step="0.01">`

### 任务 3：完善编辑功能逻辑
扩展现有编辑/保存函数，支持新增字段的编辑

#### JavaScript 修改
1. **`fillAssetDetail(asset)` 函数扩展**：
   - 填充资产类别彩色标签到标题区域
   - 填充最新价（根据资产类型显示 ¥/HK$ 前缀）
   - 填充今日盈亏金额和收益率（红绿色）
   - 计算并填充持仓占比（市值/总资产市值），渲染迷你进度条
   - 检测固收类资产（deposit/wealth），显示固收字段区域，填充到期日、年化收益率，计算距到期天数
   - 为新增编辑字段设置默认值：`editMaturity`、`editRate`

2. **`enterAssetEditMode()` 无需修改**：现有逻辑已为 `.asset-editing` 类切换输入框显示，新增字段添加 `.editable` 类即可

3. **`saveAssetInlineEdit()` 函数扩展**：
   - 读取新增字段输入值：`editMaturity`、`editRate`
   - 更新 activeAssetData 对应属性
   - 重新计算距到期天数
   - 更新持仓列表对应资产项的显示
   - 重新计算总资产和持仓占比

4. **`cancelAssetInlineEdit()` 函数扩展**：
   - 重置新增字段输入框值为原始数据
   - 调用 `fillAssetDetail()` 刷新显示

5. **添加总资产计算辅助**：
   - 新增或复用 `calculateTotalAssets()` 函数计算总资产市值，用于持仓占比计算
   - 固收类距到期天数计算：`Math.ceil((new Date(maturity) - new Date()) / (1000*60*60*24))`

## 四、注意事项
1. **货币符号兼容**：港股显示 HK$，美股显示 $ 或 US$，A股/基金/存款理财显示 ¥，根据 data-type 或 data-code 判断
2. **持仓占比计算**：需遍历所有资产计算总市值，再计算单个资产占比，编辑成本价/数量后需重新计算
3. **日期格式处理**：到期日使用 `YYYY-MM-DD` 格式输入，显示时可格式化，距到期天数需处理已过期情况
4. **字段显示控制**：非固收类资产需隐藏到期日、年化收益率、距到期天数字段，使用 `.fixed-income-only` 类名控制
5. **进度条样式**：持仓占比迷你进度条复用持仓列表中 `.weight-bar` 样式，尺寸适当缩小适配明细卡片
6. **向后兼容**：保持现有编辑流程不变，仅扩展支持字段，不破坏已有功能
